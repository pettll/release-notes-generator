# release-notes-generator

Generate structured, human-readable release notes from git commit history. Parses [Conventional Commits](https://www.conventionalcommits.org/) and organises them into sections — breaking changes, features, bug fixes, etc. Outputs Markdown, plain text, or JSON.

## Why

Manually writing release notes is tedious and often skipped. This tool automates it from the git history you already have, making it easy to drop release notes into GitHub Releases, Confluence, Jira, or your deploy pipeline notifications.

## Install

```bash
pip install release-notes-generator
```

Or run from source:

```bash
git clone https://github.com/pettll/release-notes-generator
cd release-notes-generator
pip install -e .
```

## Usage

```
release-notes --from <ref> [options]

Options:
  -f, --from <ref>      Base git ref (required) — e.g. v1.1.0, HEAD~20, main
  -t, --to <ref>        Target git ref (default: HEAD)
  -v, --version <str>   Version label in output (default: value of --to)
  --format {markdown,text,json}
                        Output format (default: markdown)
  -o, --output <file>   Write to file instead of stdout
  -r, --repo <path>     Path to git repo (default: .)
  --repo-url <url>      GitHub/GitLab URL for commit + issue hyperlinks
  -h, --help            Show help
```

## Examples

**Release notes for the last 20 commits:**
```bash
release-notes --from HEAD~20 --version 1.2.0
```

**Between two tags, with GitHub links:**
```bash
release-notes --from v1.1.0 --to v1.2.0 \
  --repo-url https://github.com/myorg/myrepo \
  --output CHANGELOG.md
```

**Plain text for Slack/email notification:**
```bash
release-notes --from v1.1.0 --to v1.2.0 --format text
```

**JSON for programmatic processing:**
```bash
release-notes --from v1.1.0 --to v1.2.0 --format json | jq '.sections.feat'
```

**Use in GitHub Actions:**
```yaml
- name: Generate release notes
  run: |
    pip install release-notes-generator
    release-notes \
      --from ${{ github.event.release.target_commitish }} \
      --to ${{ github.sha }} \
      --version ${{ github.event.release.tag_name }} \
      --repo-url ${{ github.server_url }}/${{ github.repository }} \
      --output release-notes.md

- name: Update GitHub Release body
  run: |
    gh release edit ${{ github.event.release.tag_name }} \
      --notes-file release-notes.md
```

## Commit Format

Follows the [Conventional Commits](https://www.conventionalcommits.org/) specification:

```
<type>(<scope>): <subject>

[optional body]

[optional footer — BREAKING CHANGE: ...]
```

Types and their sections:

| Type | Section |
|---|---|
| `feat` | New Features |
| `fix` | Bug Fixes |
| `perf` | Performance Improvements |
| `refactor` | Refactoring |
| `docs` | Documentation |
| `build` | Build System |
| `ci` | CI/CD |
| `test` | Tests |
| `chore` | Chores |
| Any `!` or `BREAKING CHANGE` footer | BREAKING CHANGES |

Non-conventional commits are grouped under "Other Changes".

## Example Output

```markdown
# Release v1.2.0 — 2025-03-10

## BREAKING CHANGES

- **api**: remove deprecated /v1/accounts endpoint [`a1b2c3d`]

## New Features

- **orders**: add bulk order creation endpoint (closes #42) [`f4e5d6c`]
- **auth**: support OAuth 2.0 PKCE flow [`9a8b7c6`]

## Bug Fixes

- **payments**: handle timeout on Stripe webhook retry (fixes #38) [`1d2e3f4`]

## Refactoring

- **core**: extract request validation into middleware [`5a6b7c8`]

_Generated by release-notes-generator on 2025-03-10_
```

## Salesforce / Copado Usage

Works well with Copado commit conventions. Example:

```bash
# Generate notes for a sprint release
release-notes \
  --from $(git tag -l "sprint-*" | sort -V | tail -2 | head -1) \
  --to $(git tag -l "sprint-*" | sort -V | tail -1) \
  --version "Sprint 42 Release" \
  --format text
```
